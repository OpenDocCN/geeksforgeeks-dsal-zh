# 甲骨文面试体验|第 46 集(校内)

> 原文:[https://www . geesforgeks . org/Oracle-面试-体验-设置-46-校园/](https://www.geeksforgeeks.org/oracle-interview-experience-set-46-campus/)

一个很难的笔试，持续了两个小时…没想到我入围了下一轮，也就是编码轮。

他们已经为编码回合给出了两个程序，对于这两个程序，必须满足至少 3 个测试用例才能获得面试资格..我成功地运行了所有测试用例，并被选中参加面试阶段，共 3 轮。

**第一回合:-**
持续 40 分钟，

她问了我一个不经意的问题跟我说说你自己跟了，她问我我的项目跟了一个基于内部联接按概念排序的 SQL 查询，她问我 C 和 Java 有什么区别跟我说说 C++相对于 Java 的优势，跟我说说 b/w 哈希表和哈希表的区别…然后为了测试我是否有任何编程技巧，她让我为回文写并解释一个程序。我写并解释了她让我写一个程序，用 O(n)在数组中寻找一个重复的数，为此我给出了 3 个解，她也给出了另一个解..她问我“哎呀”的概念，接着是橙色和苹果的逻辑问题，我无法解决，但她喜欢我的方法，并向我解释了答案…

**第 2 轮:-**
这持续了 20 分钟…虽然是技术轮他也问了我 HR 问题
他问我“为什么是工程？”、“为什么是 IT”、“为什么是甲骨文”？
他问:“如果你得到这份工作，合并或扩展你的技能，你生活中会发生什么样的个人变化？”
我已经对所有这些问题给出了自己的解释……
然后他让我在 C 中不用 printf 打印一些东西然后我用 OS 系统调用 write(1，“abcd”，4)然后他又问了另一个问题
Arr = {10，20，3000，9999，200 }；
打印一个最大零数的数字……
我给了 O(n2)并想进一步减少，看到我的实现，让我离开面板，准备下一轮…

**第 3 轮:-**
他问了我一些 HR 的问题和关于我家庭的问题，并让我对 java 进行自我评价……虽然我的 Java 很好，但我说 6.5，他问我“如果你的 Java 很好，那为什么是 6.5？”..然后我给了他我对 Java 的看法，他对那一层很满意，他开始问逻辑问题
1-有一辆车，一个小偷偷走了它的一个轮胎和螺母，你只有一个 Stepney，没有螺母，你怎么把它固定在车上？

2-有一个蛋糕(任何形状)的一面(或一角)受损..你如何把它切割成相等的部分(相等的数量)…

3-一根蜡烛需要 60 分钟融化…给你 2 根同等长度的蜡烛，一个火柴盒，没有提供时钟或刻度…U r 要求显示 45 分钟融化那些蜡烛
从上面的问题来看，我只回答了一个问题，我想他看到了我解决这些问题的方法，给了我很好的反馈…

我学到的最主要的一点就是脸上要保持一点微笑，对自己说的话要有自信，即使你有一点知识也要正确表达出来。
